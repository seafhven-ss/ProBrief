R-Retail-Planning-01

规则名称：经营模型先于风格表达

所属场景
零售门店

所属阶段
规划阶段

规则类型
风险规则 / 缺失信息规则 / 提案切入规则

风险等级
高

展示优先
是

规则描述
零售门店前期如果优先讨论风格、材料、视觉调性与网红感，而没有先明确客群、品类结构、收银位置、仓储需求、试衣需求、补货路径与坪效目标，项目很容易出现“空间好看但经营效率差”的问题。门店本质上首先是经营工具，其次才是视觉载体。

典型错误模式

一上来先定“高级感”“年轻化”“网红感”

先做效果图，再补功能分区

设计稿完成后才发现收银点位置不合理

仓储不足、试衣区和展示区冲突、补货动线打架

后期再用改柜体、改动线、改功能区来补救

触发条件
满足以下任一情况可触发：

输入内容明显强调“风格、视觉、材料、网红感、高级感、好看”

输入中出现门店面积，但未提及客群、商品结构、收银、仓储、试衣、动线

项目类型为零售门店，且描述偏重空间形象而非经营目标

用户明确表示“先做出效果再说”“先看方案再考虑细节”

为什么会出问题
零售门店的布局必须服务于转化、停留、陈列、试用/试穿、补货和结算。若经营模型没有前置，后续再漂亮的视觉表达也可能建立在错误的空间逻辑之上，最终影响销售效率、顾客体验和后期运营成本。

后果表现

门店空间好看但卖货效率差

某些区域天然冷场，某些区域持续拥堵

收银、仓储、试衣、补货等功能互相冲突

视觉方案完成后仍需大规模返工

设计决策与坪效目标脱节

系统输出建议文案（风险与难点）
当前需求更强调空间气质与视觉效果，但经营逻辑信息不足。若客群、品类结构、收银与仓储动线未前置确认，后续可能出现空间好看但运营效率不足的问题。

系统输出建议文案（当前缺失信息）
建议补充以下信息：

目标客群与消费层级

商品结构与主推品类

收银位置与结算方式

试衣、仓储、补货等功能需求

入口、主陈列区与高毛利商品位置关系

关联建议动作

先梳理商品结构与功能分区

明确入口、收银、试衣、仓储和补货路径

形成基础经营逻辑草图后，再进入视觉方案

若为商场店，补充商场规则与营业动线约束

关联提案切入方向

空间体验优化

展示引流

控预算高效率

页面映射模块

R-Retail-Design-05

规则名称：动线设计应基于顾客行为而非纯视觉想象

所属场景
零售门店

所属阶段
设计阶段

规则类型
风险规则 / 关键需求拆解规则 / 提案切入规则

风险等级
高

展示优先
是

规则描述
零售门店的动线设计不应只依据“画面完整”“视觉对称”“看起来有秩序”等主观审美，而应围绕真实顾客行为来组织。入口吸引区、停留点、主推商品区、引流商品区、试衣区、收银区之间的关系，必须先于纯形式表达。

典型错误模式

设计师按画面好看安排通道与中岛

顾客主路径与主推商品错位

收银线过深或过隐蔽

热销区被安排在天然冷区

试衣与主陈列区相互干扰

动线很“美”，但顾客不按设计路线走

触发条件
满足以下任一情况可触发：

输入中提到“高级感、整齐、对称、沉浸感”等视觉导向词

未明确说明入口策略、主推品区、顾客停留路径

提到商场店、集合店、精品店，但未描述试衣、收银、陈列逻辑

需求中出现“希望空间看起来很满/很丰富/很有层次”，但无行为路径信息

为什么会出问题
顾客不会因为设计师的构图逻辑而自然行动。若动线设计与顾客行为、视线、停留和购买转化逻辑脱节，空间就会出现“图上很完整，现场不转化”的问题。

后果表现

顾客绕开重点区域

部分区域拥堵，部分区域无人停留

主推商品曝光不足

体验区、试衣区、收银区关系混乱

门店表面丰富，实际转化效率差

系统输出建议文案（风险与难点）
当前需求重视视觉效果，但顾客行为路径信息不足。若入口、停留区、主推区与收银线关系未基于真实使用行为梳理，后续可能出现空间完整但转化效率偏低的问题。

系统输出建议文案（关键需求拆解）
建议将动线相关需求拆解为：

入口吸引策略

主陈列区与引流区关系

顾客主停留路径

试衣与收银动线

补货与后台动线分离需求

关联建议动作

先画顾客行为动线草图

标记高毛利商品区与引流商品区

确认试衣、陈列、收银三者关系

对主路径与拥堵点进行预判

关联提案切入方向

展示引流

空间体验优化

品牌调性塑造

页面映射模块

关键需求拆解

R-Retail-Design-08

规则名称：预算必须拆项并设置优先级，不能只看总价

所属场景
零售门店

所属阶段
设计阶段

规则类型
成本规则 / 风险规则 / 下一步建议规则

风险等级
高

展示优先
是

规则描述
门店项目若只讨论“总预算大概多少”，而没有进一步拆分为硬装、定制、灯光、机电、门头、软装、设备、不可预见费等具体项，并明确删减优先级，后续一旦超支，项目会进入无序删减状态，最终牺牲的往往是最影响体验与运营的部分。

典型错误模式

客户只说“预算大概 40 万左右”

设计阶段直接按理想方案推进

中后期发现超支后临时砍项

先砍灯光、收银、仓储、材质层次等关键体验项

预算删减没有顺序，结果既不省钱也不成体系

触发条件
满足以下任一情况可触发：

出现“预算中等、预算有限、控制预算、不要太贵”等描述

预算字段只有总额，没有拆项说明

输入中强调效果与高级感，同时预算表达模糊

用户明显希望“效果强”且“成本低”

为什么会出问题
总价预算无法支撑设计阶段的取舍。没有拆项和优先级，就无法判断哪些内容是必须保留的经营与体验核心，哪些可以作为弹性项。后期一旦进入删减，决策会变得被动且失序。

后果表现

设计与造价严重脱节

中后期大面积返工或删改

关键体验项被迫牺牲

项目整体感下降

客户对成本失控和“效果缩水”不满

系统输出建议文案（风险与难点）
当前预算信息更接近总价表达，缺少拆项与优先级。若方案阶段未明确硬装、定制、灯光、设备及预备金结构，后续一旦超支，容易出现无序删减并影响核心体验。

系统输出建议文案（当前缺失信息）
建议补充：

硬装预算范围

定制柜体与陈列道具预算

灯光与机电预算

门头与品牌视觉预算

软装与设备预算

不可预见费比例

必保项 / 可调整项排序

关联建议动作

将预算拆解为主要成本模块

明确“必须保留项”和“可弹性项”

在概念方案阶段同步做分档报价

对高成本定制项提前评估替代方案

关联提案切入方向

控预算高效率

快速落地

标准化复制

页面映射模块

当前缺失信息

风险与难点

R-Booth-Planning-13

规则名称：展馆规则先于造型创意

所属场景
展会展台

所属阶段
规划阶段

规则类型
合规规则 / 风险规则 / 缺失信息规则

风险等级
高

展示优先
是

规则描述
展台项目的创意方案必须建立在展馆规则基础上，包括限高、吊点、双层限制、全封顶限制、报馆要求、审图要求、材料限制、进场时段等。展馆规则不是后补条件，而是设计前提。

典型错误模式

先追求炸裂造型，再查限高

方案做了吊挂结构，但未确认吊点条件

做双层或大体量结构后才发现报馆限制

全封顶造型未考虑消防与审图要求

方案很强，但无法通过审图或进场执行

触发条件
满足以下任一情况可触发：

输入出现“科技感、震撼、大气、吊装、超高、双层、封顶”等表达

未明确展馆规则、展位尺寸、限高、开口面情况

用户强调视觉冲击力，但没有规则约束信息

输入是展台项目，且缺少报馆条件描述

为什么会出问题
展台不同于普通空间设计，其施工窗口短、报馆程序严格、安全约束强。若创意先行而规则缺失，后续最容易出现方案被迫重做、审图不通过、施工受限甚至现场被叫停。

后果表现

设计方案无法报审

创意方案大幅缩水

工期被动压缩

成本上升

现场临时调整风险增加

系统输出建议文案（风险与难点）
当前需求强调展台表现力，但展馆规则信息不足。若限高、吊点、报馆要求与结构限制未前置核对，后续方案可能出现创意强但无法落地的问题。

系统输出建议文案（当前缺失信息）
建议补充：

展馆与展会名称

展位尺寸与开口面

限高与吊点条件

是否允许双层或吊挂

是否存在全封顶限制

报馆与审图时间节点

关联建议动作

先核对展馆规则与参展商手册

明确展位尺寸、限高、开口面和吊点

在概念方案前确认可做范围

将规则约束纳入第一轮设计边界

关联提案切入方向

快速落地

控预算高效率

展示引流

页面映射模块

当前缺失信息

风险与难点

R-Booth-Planning-14

规则名称：审图是项目里程碑，不是形式流程

所属场景
展会展台

所属阶段
规划阶段

规则类型
合规规则 / 进度规则 / 风险规则

风险等级
高

展示优先
是

规则描述
展台项目中的审图不是简单的行政流程，而是决定后续报馆、进场与施工是否成立的关键里程碑。效果图、平面图、施工图、材料说明、消防/电路图及特殊结构审核，都是展台项目进度计划的一部分。

典型错误模式

把审图当成后补材料

设计阶段只出效果图，不同步准备报审资料

图纸颗粒度不足，审图被退回

审图节点没有纳入总工期，导致后续时间被压缩

审图通过前就启动采购或制作，形成被动

触发条件
满足以下任一情况可触发：

展台项目未说明审图时间、报馆流程或图纸准备情况

输入强调工期紧、开幕时间近

方案涉及特殊结构、吊挂、双层、超高

客户将审图理解为“后面补一下”

为什么会出问题
展台项目时间高度压缩，审图一旦延误，会直接压缩制作与进场时间。若审图资料不完整或节点未前置管理，整个项目节奏就会变得极为被动。

后果表现

审图退回，后续延误

已定方案被迫调整

采购与制作被中断

临展前期时间极度紧张

现场风险与加班成本上升

系统输出建议文案（风险与难点）
当前项目时间信息存在压力，但审图节点尚未明确。对于展台项目，审图本身属于关键里程碑，若未纳入前期倒排计划，后续制作与进场时间将明显受压缩。

系统输出建议文案（下一步建议）
建议优先完成：

确认报馆与审图截止日期

明确审图资料清单

对特殊结构提前进行结构审核准备

将审图通过时间设为制作启动前置节点

关联建议动作

建立审图专项清单

审图资料与设计同步推进

对特殊结构提早做技术核对

将审图结果作为制作启动门槛

关联提案切入方向

快速落地

风险前置管理

高压工期闭环执行

页面映射模块

风险与难点

R-Commercial-Design-25

规则名称：概念表达必须同步技术落地条件

所属场景
商业空间 / 展厅 / 品牌空间 / 综合商业空间

所属阶段
设计阶段

规则类型
风险规则 / 缺失信息规则 / 提案切入规则

风险等级
高

展示优先
是

规则描述
商业空间设计中，概念表达若与机电、结构、消防、设备、维护、材料可实施性脱节，就会形成“概念强、落地弱”的典型风险。越是复杂空间，越不能在深化阶段才引入技术约束。

典型错误模式

概念方案先行，技术条件后补

异形结构、复杂天花、特殊材料在效果图中成立，现场难以实现

消防点位、机电需求、设备检修空间未纳入设计

维护与清洁成本未考虑

最后由施工阶段被动修正概念

触发条件
满足以下任一情况可触发：

输入强调“品牌感、艺术感、沉浸感、概念感、强视觉表达”

未明确机电、消防、设备、维护条件

出现异形结构、复杂材料、整合性展示要求

项目类型为展厅/品牌空间/综合商业空间，且描述明显偏概念导向

为什么会出问题
商业空间通常涉及更多系统协同，技术限制并不是后期校正项，而是设计的一部分。若概念与技术条件脱节，后续会发生返工、删减、妥协，甚至使核心表达失真。

后果表现

概念方案无法完整落地

深化阶段大规模改动

工期与预算被动增加

维护与运营成本被低估

设计表达与最终落地偏差大

系统输出建议文案（风险与难点）
当前需求强调空间表达与品牌感，但技术落地条件信息不足。若机电、结构、消防、设备与维护条件未同步进入设计，后续可能出现概念强但落地代价过高的问题。

系统输出建议文案（当前缺失信息）
建议补充：

机电与设备需求

消防与结构限制

维护与检修空间要求

关键材料可实施性

复杂节点的技术边界

关联建议动作

在概念阶段同步引入技术约束

对复杂节点做提前校核

对高风险表达做样板或替代方案预判

将维护成本纳入设计判断

关联提案切入方向

品牌调性塑造

快速落地

控预算高效率

页面映射模块

当前缺失信息

R-Cross-Schedule-31

规则名称：江浙沪皖赣特殊季节会显著拉长施工周期

所属场景
零售门店 / 商业空间 / 部分展台预制环节

所属阶段
规划阶段 / 施工阶段

规则类型
进度规则 / 风险规则 / 展示型规则

风险等级
中高

展示优先
是

规则描述
在江苏、浙江、上海、安徽、江西等区域，部分月份的自然环境会显著影响施工节奏。尤其是 6 月梅雨季，以及 12 月、1 月、2 月低温季，乳胶漆、涂料、基层干燥、局部固化与部分安装工序通常会比正常月份更慢，实际工期常显著长于常规预估。

典型错误模式

仍按常规月份安排工期

涂料、基层、油漆等工序未预留缓冲

对天气、湿度、低温导致的固化变慢缺少预判

开业或交付节点定得很紧，未设置季节性余量

最终出现“看似施工正常，但总进度持续被拉长”

触发条件
满足以下任一情况可触发：

项目地区位于江苏、浙江、上海、安徽、江西

计划施工或交付时间落在 6 月、12 月、1 月、2 月

规则名称：高造价特制定制件会显著拉长周期并放大协同风险

所属场景
零售门店 / 商业空间 / 展会展台

规则类型
进度规则 / 成本规则 / 风险规则 / 提案切入规则

风险等级
高

展示优先
是

规则描述
造型雕塑、玻璃钢制品、烤漆制品、超长超宽玻璃、复杂异形定制件等高造价特制定制内容，通常同时受造型复杂度、加工周期、天气环境、运输条件、现场安装精度和返修成本影响。一旦项目强调高效推进，这类内容往往会成为工期和协同风险源。

典型错误模式

方案阶段大量采用复杂特制定制件

只看到效果，不评估生产与安装难度

特制定制件与总工期并行推进但无缓冲

后期发现生产周期超预期

安装误差、返工、运输与天气进一步放大延误

触发条件
满足以下任一情况可触发：

输入中出现“雕塑、玻璃钢、烤漆、异形、超长玻璃、特殊装置、定制艺术装置”等词

需求强调高端效果、强记忆点、独特装置

同时又提出工期快、预算控制、快速交付

项目位于天气敏感月份或高压施工周期内

为什么会出问题
特制定制件的风险不只在造价，更在于其不可替代性和不确定性。一旦生产、运输、安装任一环节出问题，替代方案少，补救成本高，极易拖累整体进度。

后果表现

项目周期被显著拉长

特制定制件成为关键路径阻塞点

安装与运输风险加大

成本超支

为赶时间被迫换材或降低实现度

项目整体节奏失控

系统输出建议文案（风险与难点）
当前需求若采用较多特制定制件，虽然可提升空间识别度，但也会显著增加生产、运输、安装与返修的不确定性。对于强调高效推进的项目，这类内容可能成为工期与成本的主要风险来源。

系统输出建议文案（提案切入建议）
若项目更重视交付效率与预算控制，建议优先采用标准化模块、常规工艺或可替代材料，减少高不确定性的特制定制内容。

对高造价定制件做清单化评估
