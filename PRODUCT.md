# Brief Builder — 产品说明

> 给 Nexus 出文案用的产品素材文档。包含产品定位、核心卖点、技术亮点、目标用户画像和使用场景。

---

## 一句话定义

Brief Builder 是一个 AI 驱动的提案结构化工具——你描述项目需求，它输出一份专业的、可直接用于商务沟通的 Brief 文档。

## 产品定位

不是文案生成器，不是 ChatGPT 套壳。

Brief Builder 解决的是一个具体的工作流问题：**把模糊的项目需求变成结构化的提案文档**。它的输出不是"一篇文章"，而是一份包含需求拆解、缺失信息、风险评估和下一步行动的工作文件。

核心理念：**AI 不替你做决策，而是帮你把该想到的都想到。**

## 谁需要这个工具

| 角色 | 场景 | 痛点 |
|------|------|------|
| 自由设计师 | 接到客户需求，要出提案 | 每次从空白文档开始，容易遗漏关键问题 |
| 项目经理 | 整理需求给团队 | 客户说的模糊，要反复确认才能推进 |
| 创业者 | 有想法要落地 | 脑子里一团东西，不知道从哪开始 |
| 乙方团队 | 投标/竞标准备 | 需要快速出结构化方案，抢时间窗口 |
| 品牌/市场人员 | 策划活动或空间项目 | 涉及多个供应商，需要统一的 brief 格式 |

## 核心功能

### 1. 表单化输入
填写项目类型、描述、预算、时间线，不需要写 prompt，不需要 AI 经验。

### 2. 结构化输出（6 个模块）
- **项目摘要** — 自动提炼标题和概述
- **关键需求** — 按优先级拆解（高/中/低）
- **缺失信息** — 标注遗漏项及其影响等级（阻塞/重要/可选）
- **风险提示** — 识别潜在问题及严重程度
- **下一步行动** — 具体步骤 + 负责方
- **提案切入角度** — 给出商务沟通的策略建议

### 3. 行业规则引擎
不完全依赖 LLM。内置零售门店、展会展台、商业空间等行业的专业规则库，确保输出中包含行业常识性的检查项（比如零售项目会自动检查是否考虑了收银动线和仓储需求）。

### 4. 冲突过滤
如果你已经在描述中提到了预算，AI 不会再把"预算未知"列为缺失信息。系统会提取你输入中的事实，过滤掉矛盾项。

### 5. 一键导出
复制到剪贴板或下载为文本文件。

## 技术亮点（面向开发者/技术内容）

### 混合架构：规则引擎 + LLM
不是纯 LLM 生成。采用 **确定性规则 + AI 推理** 的混合管线：

```
用户输入 → 事实提取 → 规则引擎匹配 → LLM 生成 → 冲突过滤 → 安全护栏 → 结构化输出
```

- **事实提取**（Facts Extraction）：从自然语言中提取 VI 状态、预算清晰度、面积、时间线等结构化事实
- **规则引擎**（Rule Engine）：8+ 条行业规则，基于项目类型、关键词、地区、月份等维度触发
- **LLM 层**：支持 Groq（qwen3-32b）和 OpenAI（gpt-4.1-mini）双通道，JSON Schema 约束输出
- **冲突过滤**（Conflict Filter）：用提取的事实校验 LLM 输出，移除矛盾项
- **场景隔离**（Scene Isolation）：零售/展会/商业空间各有独立 prompt，防止跨场景污染

### 为什么不全用 LLM

1. **可预测性** — 行业规则的输出是确定性的，不会因 prompt 微调产生意外结果
2. **可审计** — 每条规则有 ID，可以追溯触发原因（调试面板可见）
3. **低延迟** — 规则匹配几乎零延迟，LLM 只负责自然语言生成
4. **低成本** — 规则层处理了大量确定性判断，减少 LLM token 消耗

### 技术栈
- Next.js 16 + React 19 + Tailwind CSS 4 + TypeScript 5
- 后端 API Route 处理 LLM 调用
- 零外部依赖（除 Next.js 核心），约 1600 行源码

## 当前版本状态

**v0.1 — MVP**

- 6 种项目类型（零售门店/展会展台/品牌快闪/商业空间升级/办公展示空间/其他）
- 8+ 条行业规则
- Groq + OpenAI 双 LLM 支持
- 中文界面
- 调试面板（规则匹配 + 事实提取 + 过滤日志）

## 未来方向

- 多语言支持（英文界面 + 英文输出）
- 自定义规则（用户添加自己行业的检查项）
- Brief 历史记录
- Markdown / PDF 导出
- 团队协作（分享链接、评论）
- 更多行业模板（餐饮、教育、医疗等）
